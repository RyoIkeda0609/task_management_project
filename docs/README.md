# README

## docsディレクトリの構成

このディレクトリは以下の3ディレクトリを持つ

- spec
- prompt
- ard
- memo

## spec ディレクトリの使い方

### 目的

- 最終的な成果物を配置するためのディレクトリ
- 基本的にMarkdown形式で、開発者やAIが参照するためのドキュメントを配置する

### 使い方

- 成果物をこちらに配置するため、必ずGitでのバージョン管理対象とする
- ClaudeCode等のAIが記載してくれたドキュメントを配置するため、開発者以外の変更が加わることも想定しておく
- 最終的にだれが見てもわかるように、構造化して、あいまいさを避けた文章を心掛ける

## prompt ディレクトリの使い方

### 目的

- AIと壁打ちするための質問事項や、その回答を記載しておくためのディレクトリ
- 基本的にMarkdown形式で、私がAIに聞くためのメモ、その回答を配置する

### 使い方

- 過去の履歴をこちらに配置するため、Gitでのバージョン管理対象とすることをすすめる
- 誰が書き換えるかわからないため、最終的な成果物は配置しないようにする
- 適当な文章で構わないが、作業中のドキュメントであることを忘れないようにする

## ard ディレクトリの使い方

### 目的

- コーディング規約的なもの等を記載する

### 使い方

- コミットの対象とする

## memo ディレクトリの使い方

### 目的

- 上記以外のその他重要なことや一時保存的な情報を配置するディレクトリ
- Markdown以外の形式を配置することもあり、広く参照される想定はない作業ディレクトリとしての役割

### 使い方

- WIPのファイルを配置する目的のため、Gitのバージョン管理対象外とする
- 誰が書き換えるかわからないため、最終的な成果物は配置しないようにする
- あくまでメモのため、参照はほどほどにする
