# プロダクト概要メモ（ドラフト）

## 1. プロダクトの前提

- 種別：個人向けモバイルアプリケーション
- プラットフォーム：iOS / Android（Flutter）
- 想定ユーザー：
  - 目的・目標はあるが、日々の行動に落とし込めていない個人
  - タスク管理アプリは使ったことがあるが、形骸化している人

- 開発動機：
  - 開発者自身が使いたい
  - 同様の課題を持つユーザーは多いという仮説

## 2. 収益モデル（暫定）

- 買い切り型をメインとする
- 想定価格：1,000円
- 目標売上：月100万円
  - 1,000円 × 1,000インストール / 月

※ サブスクリプションや追加課金は現時点では検討対象外（将来検討）

## 3. 競合・代替手段

- 主な利用経験：Notion

### Notionに対する不満点

- 自由度が高すぎる
- タスク・目標・構造をすべてユーザー自身で設計する必要がある
- 結果として
  - 目的とタスクの関係が曖昧になる
  - 継続利用が難しい

## 4. 解決したい本質的な課題

### 現状のタスク管理アプリの偏り

- 単発タスクの管理が中心
- メモ用途に近い
- ゴール・目的との関連が弱い

### 問題

- 何のためのタスクか分からなくなる
- ゴールから逆算した計画が立てられない
- 目的達成までの道筋が見えない
- 自分で構造化するのが面倒

### 本質的な課題

- 「目的 → 計画 → 行動」への分解を人力で行うのが難しい

## 5. プロダクトコンセプト（中核）

### 一言で表すと

**「目的達成のために、今日やることが分かるアプリ」**

### 代表的ペルソナ

- 高校2年生
- 大学受験を控えており、志望校は決まっている
- しかし以下の状態に陥っている：
  - 志望校合格のために「何を」「どの順番で」やればいいか分からない
  - 点数・科目・勉強内容・日々の行動が頭の中で整理できていない
  - 計画を立てようとしても、複雑になり挫折してしまう

### ペルソナの本質的な悩み

- 何を優先すべきか分からない
- ゴールから逆算できない
- 行動と目的が結びついていない

### 基本構造（思想）

- ゴール（最終目的）
  ↓
- マイルストーン（目標）
  ↓
- タスク（日々の行動）

この階層構造を**アプリ側が唯一の正解として提供**する。

### 狙い

- 目的から逆算する思考を強制的に体験させる
- 構造化をユーザーに任せず、アプリが肩代わりする

### やらないこと（思想を守るための制限）

- メモ帳化しない
- 自由な階層構造は提供しない
- チーム共有・チーム利用は行わない
- SNS機能は入れない
- 複雑な UI / UX は採用しない
- ログイン機能は必須としない
- 共有機能・外部サービス連携は必須としない

## 6. 提供したい体験（コア価値）

- ゴールを決めると、やるべきことが自然に分解される
- 今日やるべきことが即座に分かる
- いまの行動が、どのゴールにつながっているか理解できる
- 日々の達成が、未来を近づけている実感につながる

### 1か月後にユーザーが到達してほしい状態

- 目的を明確に言語化できている
- ゴール → 目標 → 行動を自分で整理できる
- 不安ではなく「前に進んでいる」という実感を持っている

## 7. 想定ビュー（例）

※ ペルソナ（大学受験生）が直感的に理解できることを最優先とする

- ゴールピラミッドビュー
  - ゴール → マイルストーン → タスクを階層的に可視化

- ゴール達成カレンダービュー
  - ゴール期限を起点に、
    - マイルストーン期限
    - タスク実行日
      を時間軸で確認

- 今日のタスクビュー
  - 複数ゴールを横断して
  - 「今日やるべきタスク」だけを一覧表示

## 8. 絶対に負けないポイント

- わかりやすい UI / UX
- 構造化思考のしやすさ
- ゴール達成に特化した体験設計
