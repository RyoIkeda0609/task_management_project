# UIUXの改善

- 各ゴールのカレンダービューでスクロールができないのを改善したい
  - 現状：カレンダーとタスクが並んでいるが、タスクの一覧のみスクロール可能で、カレンダー部分が固定されており、タスクの一覧が見づらい
  - 理想：カレンダーの部分含めてスクロール可能としたい
- 本日のタスクにで表示できるのを「本日以外」も含めたい
  - 現状：閲覧日当日のタスクのみ閲覧可能
  - 理想：日付切り替えのパーツで日付を切り替えると、該当の日付が期限のタスクが一覧表示される
- ホームのフィルターのリネームをしたい
  - 現状：「活動中」「完了済み」
  - 理想：「進行中」「完了」
- ホームのゴール一覧の並び順を変更できるようにしたい（ドロップダウンで並び順変更機能のイメージ）
  - 現状：期限が近い順のみ
  - 理想：「期限が近い順」「進捗の良い順」「進捗の悪い順」「カテゴリ別」
    - さらに理想（ロジックに大幅な修正が加わらないのであれば）：「マイルストーンの多い順」「マイルストーンの少ない順」「タスクの多い順」「タスクの少ない順」
      - これはPresentation層の修正でロジックをゴリゴリ組んだり、今まで使っていなかったProviderからデータを取ったりする必要があれば劣後でよいです（ロジックはできるだけシンプルなままで機能追加できるもののみしたいため）
- 作成したゴールがすぐにゴール一覧に表示されない問題を解消したい
  - 現状：ゴール一覧からゴール作成→作成完了モーダル→ホームに自動で戻る→作ったゴールがまだない→ホットリロード→作ったゴールが表示される
  - 理想：ホームに自動で戻る→作ったゴールが表示される
- 編集したゴールがすぐにゴール一覧に表示されない問題を解消したい
  - 現状：ゴール詳細からゴール編集→編集完了モーダル→ゴール詳細に自動で戻る→ホームに戻る→編集したゴールがまだ反映されていない→ホットリロード→編集後のゴールが表示される
  - 理想：ホームに戻る→編集後のゴールが表示される
- タスクのステータス変更がゴール一覧に反映されていない
  - 現状：本日のタスク（タスク詳細）から変更したタスクの状態がゴール一覧に反映されず、ホットリロードで反映される
  - 理想：常に最新の状態を表示したい（難しければ再取得ボタン・更新ボタンみたいなものを置くのでもよい）
- また、上記以外にも不具合がないか合わせて確認してほしい
