# エミュレータテストでの修正点（ステップ1）

## 不具合・修正点

- オンボーディング画面
  - 起動時に毎回出る気がする
    - 期待値：アプリ初回起動時には表示されるが、2回目以降は表示されない
  - ボタンの位置が不ぞろい（画面1から画面2へ行く際のボタンが押しづらい（間違えて「戻る」を押してしまいやすいため、「戻る」ボタンがいらないかもしれない））
    - 期待値：画面1→画面2→ホームへスムーズに遷移できる
- ホーム（ゴール一覧）
  - リスト形式
    - 情報が少ないのでもう少し追加したい
      - カテゴリ
      - 進捗
    - 並び順が不明（作成日時の降順とかがわかりやすいかもしれない）
  - ピラミッド形式
    - こちらもリスト形式同様、情報を追加したい
- ゴール詳細画面
  - 詳細画面でピラミッドビューになるが、マイルストーンが1つしかないときにゴールのオブジェクトが真ん中に表示されて少し変に見える
  - マイルストーン追加画面に遷移すると、そこから戻ることができない（Another exception was thrown: GoError: There is nothing to pop）
  - 「このゴールに紐づけます」というのにゴールのIDが表示されていて、ユーザーは何のことかわからない（わかりやすい表示にしたい）
- マイルストーン詳細画面
  - 編集、削除の導線がなく、操作ができない
  - 「タスクを追加」ボタンを教えても反応しない
  - 「タスクを追加」ボタンが横に長すぎる（画面いっぱい過ぎる）
  - タスクのステータスを押してもステータスを更新できない（3点リーダーからしか変更できないが、直感的ではない）
  - タスクの削除を行うと、削除の確認モーダルが消えずに画面に残ってしまう
- タスク作成画面
  - タスクの説明が画面上は任意と書いてあるが、入力しないとエラーが発生する
  - マイルストーンに紐づけられますと書いてあるが、マイルストーンIDが表示されていて、ユーザーは何のことかわからない
  - キャンセルを押すと、黒い画面に遷移する
- タスク詳細画面
  - ステータスの更新ボタンが多い（ステータス更新は todo -> doing -> done -> todo なので、それぞれのステータスに変更するボタンはいらず、ステータスを更新するだけでよい（勝手に上記のように循環してくれればそれでよい））
  - タスク詳細画面から戻ると、本日のタスクに遷移してしまう（本当は開いた画面に戻りたい（マイルストーンから開けばマイルストーンへ、本日のタスクから開けば本日のタスクへ戻す））
  - 編集、削除の導線がなく、操作できない
  - タスク情報にタスクIDとマイルストーンIDが表示されているが、ユーザーには何のことかわからない
- 本日のタスク
  - 本日のタスクの進捗がおかしい（todo:0%, doing:50%, done:100%のルールが守られておらず、Doneしか進捗に反映されていない）
  - 本日のタスクから詳細に遷移できない（タスクのどこを押してもステータス変更になってしまい、詳細に遷移できない、ステータスの場所を押したときはステータスを更新して、そうでないときは詳細に遷移させたい）
