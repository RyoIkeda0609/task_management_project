# 前提

- あなたは優秀なエンジニアです。
- 私はマネージャーとしてこのアプリの作成をリードしています。
- 現在タスク管理アプリを作成しています。
- アプリ名は「タスクピラミッド」
  - アプリでは、ゴール、マイルストーン、タスクの順で作成します。
- ゴールから逆算して日々のタスクを考えてもらうのがこのアプリの目標です。

# 機能

- 前提の通り、このアプリでは、ゴール、マイルストーン、タスクを用意します。
- これらはすべてアイテムという親クラスを持ち、以下の共通のプロパティを持ちます。
  - ID（自動採番）
  - タイトル（ユーザー入力、必須、100文字まで）
  - 説明（ユーザー入力、非必須、500文字まで）
  - 期限（日付、ユーザー入力、必須、明日以降のみ）
- これらのアイテムは、すべて以下の画面を持ちます。
  - 一覧
  - 詳細
  - 作成
  - 編集
- 画面の依存関係は以下の通りです。現在の実装でこの機能を実現するのに抜け漏れはないですか？
  - ゴール一覧
    - ゴール詳細
      - ゴールの詳細が見れる
      - タイトル、説明、期限、カテゴリ、進捗、そのゴールに紐づくマイルストーン一覧が確認できる
      - ゴール編集、ゴール削除の導線がある
      - ゴール編集
        - ゴール詳細のAppBarの右上部分から遷移できる(削除ボタンの左)
        - タイトル、説明、期限、カテゴリの編集ができる
        - 編集が成功すると、編集完了モーダルを表示し、ゴール一覧に自動で戻る
      - ゴール削除
        - ゴールを削除することができる
        - ゴール詳細のAppBarの右上部分から削除できる(編集ボタンの右)
        - ゴールを削除すると、子供のマイルストーン、その子供のタスクも削除される(カスケード削除)
      - マイルストーン一覧(ゴール詳細)
        - ゴール詳細の下部分に、そのゴールに紐づくマイルストーンが一覧表示される
        - マイルストーン作成・詳細に遷移可能
          - マイルストーン詳細
            - ゴール詳細のマイルストーンから遷移可能
            - マイルストーン編集
              - マイルストーン詳細のAppBarの右上部分から遷移できる(削除ボタンの左)
              - タイトル、説明、期限の編集ができる
              - 編集が成功すると、編集完了モーダルを表示し、マイルストーン一覧(ゴール詳細)に自動で戻る
            - マイルストーン削除
              - マイルストーンを削除することができる
              - マイルストーン詳細のAppBarの右上部分から削除できる(編集ボタンの右)
              - マイルストーンを削除すると、子供のタスクも削除される(カスケード削除)
            - タスク一覧(マイルストーン詳細)
              - マイルストーン詳細の下部分に、そのマイルストーンに紐づくタスクが一覧表示される
                - タスク作成・詳細に遷移可能
                  - タスク詳細
                    - マイルストーン詳細のタスクから遷移可能
                    - タスク編集
                      - タスク詳細のAppBarの右上部分から遷移できる(削除ボタンの左)
                      - タイトル、説明、期限の編集ができる
                      - 編集が成功すると、編集完了モーダルを表示し、タスク一覧(マイルストーン詳細)に自動で戻る
                    - タスク削除
                      - タスクを削除することができる
                      - タスク詳細のAppBarの右上部分から削除できる(編集ボタンの右)
                      - タスクを削除すると親の画面に戻る
                    - タスク作成
                      - マイルストーン一覧の画面下部およびFABから遷移可能
                      - タイトル、説明、期限の入力ができる
                      - 入力が成功すると、作成完了モーダルを表示し、タスク一覧(マイルストーン詳細)に自動で戻る
          - マイルストーン作成
            - ゴール一覧の画面下部およびFABから遷移可能
            - タイトル、説明、期限の入力ができる
            - 入力が成功すると、作成完了モーダルを表示し、マイルストーン一覧(ゴール詳細)に自動で戻る
    - ゴール作成
      - タイトル、説明、期限、カテゴリを入力
      - 作成が成功すると作成完了モーダルを表示し、ゴール一覧に自動で戻る
      - 「戻る」・「キャンセル」を押すと、ゴール一覧に戻る

# 実装

上記の機能を満たすため、このアプリの現在の実装をレビューしてほしいです。
観点は以下の通り

- クリーンアーキテクチャかどうか
  - DomainはApplication層より外に依存していないか
  - ApplicationはInfrastructure層やPresentation層に依存していないか
  - 責任の分離が正しくできているか
- クリーンコードか
  - 不要なコメントアウトがないか
  - ファイルの構造はそろっているか
  - 読みやすいコード化など
- SOLID原則を満たしているか
- バグと思われる個所はないか
