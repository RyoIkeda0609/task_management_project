# UIUXの向上

## 目的

- ユーザーをより多く獲得し、より長く使ってもらえるようなUIUXにすること

## 原則

- Presentation層のみの修正とする
- Domain,Appliaction,Infrastructureはこれまでの実装ですでに担保できていることとする
- もしできていない場合は大きな修正が必要になるため、事前に私に確認する

## 修正箇所

1. オンボーディング画面
   1. このアプリのコンセプトをもっと打ち出したい
      1. 1枚目：ゴールに関すること(ゴールとは、ゴールの考え方、ゴールの具体例)
      2. 2枚目：マイルストーンに関すること(マイルストーンとは、マイルストーンの考え方、マイルストーンの具体例)
      3. 3枚目：タスクに関すること(タスクとは、タスクの考え方、タスクの具体例)
      4. 4枚目：ゴール→マイルストーン→タスクと逆算して考えられるということ(ゴールを頂点としたピラミッドみたいだよね、マイルストーンを用意して、日々のタスクをクリアすると、目標はかなえられるよ、みたいなメッセージ)
      5. 5枚目：さあ、始めよう、みたいな感じ
   2. 現状、エミュレータでのみ確認しているが、毎回出てしまっているので、1度見た人には再度出したくない
      1. Riverpodで管理している？これを端末保存(Hiveとか)にしたら永続化できて毎回出なくなるのでは？
2. 各アイテムの画面(共通)
   1. 編集画面に入力済みの値がプリセットされていない
      1. ゴールの名称のみ変えたいときに、説明とかを入力しなおす必要があるため、入力欄には事前にプリセットしてほしい
   2. 作成ボタンはIcons.add_commentではなく、Icons.addを使ってほしい
   3. 期限にデフォルトで入力されているのが今日なので、アイテムの期限ルール(明日以降)に違反し、モーダルが出てしまうので、最初から初期値を明日にしておきたい
   4. バリデーションエラーモーダルのキャンセルボタンがやや不自然な位置にあるので、修正したい
3. マイルストーン編集画面、タスク作成・編集画面
   1. 以前に入力途中でキャンセルしたものが、次の入力に出てしまっているため、キャンセルした場合、保存や変更しないようにしたい
   2. 類似の事象があげた個所以外に起きないかも確認してほしい
4. 下タブの挙動
   1. どうしてもタスクを作成したりすると、階層が深くなってしまうので、ホームボタンを押すと、その画面の1番トップに遷移したい(タスク詳細でホームを押すと、ホーム画面に戻る、ほかの下タブも同様に、その下タブの一番上に戻る)
5. 全体的にアプリを見て、違和感のあるUIUXになっていないかを確認してほしい
   1. よりよくできる箇所があれば提案してほしい
   2. ユーザーが毎日使いたくなるようなレイアウトにしたい
