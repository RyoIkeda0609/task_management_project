# Presentation Polish Playbook

Presentation を 100 点構造へ持っていくための実行手順

目的：
現在 70〜85 点レベルの UI を
再現可能な工業規格へ引き上げる。

---

# 🎯 完成形の定義

- どの画面も同じ匂い
- build を読めば構造が一瞬で分かる
- UI が判断していない
- ViewModel が肥大しない
- 迷いがない

---

---

# 🔥 まず理解すること

全部一気に直すと失敗する。

画面単位で、
段階的に完成させる。

---

---

# ====================================

# Phase 1

# Page から知恵を抜く

# ====================================

---

## 症状

Page / build 内に

- 三項演算子
- 表示前加工
- 文言生成
- list の並び替え

が存在する。

---

## やること

それらを

→ ViewModel  
→ もしくは State

へ移動。

---

## 目標

Page = 配線係。

---

---

# ====================================

# Phase 2

# build を型に入れる

# ====================================

---

## すべての画面を以下へ強制

return Scaffold(
appBar: ...
body: \_Body(),
floatingActionButton: ...
);

---

Body の中身：

Header
Content
Footer or Action

---

迷わせない。

---

---

# ====================================

# Phase 3

# if の駆逐

# ====================================

---

## 見つけたら即やる

if (loading)
if (error)
if (empty)

---

→ それぞれ Widget へ。

---

### 例

Bad:
if (state.isLoading) return Circular...

Good:
return LoadingView();

---
