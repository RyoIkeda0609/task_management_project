## 2. アプリの概要

### 2.1 プロダクト名

**タスクピラミッド**(ゴールを頂点とし、マイルストーン、タスクを整理することで、一歩一歩タスクを進めればいずれゴール(目標)がかなうとするもの)

### 2.2 何をするアプリか

「ゴール → マイルストーン → タスク」の **3階層構造を強制** することで、
ユーザーが **目的達成のために、今日やるべきことを迷わず実行できる** ようにする個人向けタスク管理アプリケーションです。

### 2.3 一言で表すと

> **「目的達成のために、今日やることが分かるアプリ」**

---

## 3. プロダクトの目的

### 3.1 解決する課題

既存のタスク管理アプリの多くは自由度が高く、構造設計をユーザーに委ねています。
その結果、以下の問題が起きます。

- 何のためのタスクか分からなくなる
- ゴールから逆算した計画が立てられない
- 行動が継続しない

**本質的課題**：「目的 → 計画 → 行動」への分解を人力で行うことは難しい

### 3.2 このアプリが提供する解決策

- **構造の強制**：ゴール → マイルストーン → タスクの3階層のみを許可し、構造設計をユーザーに委ねない
- **今日やることの明確化**：複数ゴールを横断して「今日やるタスク」を一覧表示する
- **進捗の自動可視化**：タスク完了に基づく進捗を自動計算し、努力の積み上げを実感させる

## 4. プロダクトコンセプト

### 4.1 コア思想（最上位原則）

本プロダクトのすべての意思決定は、以下の4原則に従います。

| No  | 原則                                     | 意味                                                 |
| --- | ---------------------------------------- | ---------------------------------------------------- |
| 1   | **構造は強制、分解はユーザー**           | 3階層構造は唯一の正解。ユーザーに構造設計を委ねない  |
| 2   | **思考を代行しない**                     | AIによる自動分解やテンプレートでの思考省略をしない   |
| 3   | **自由度を増やさないことで迷いを減らす** | 機能追加は「便利さ」ではなく「迷いの排除」で判断する |
| 4   | **ゴール達成体験を最優先する**           | すべてのUI/UX判断はゴール達成に近づく体験を優先する  |

### 4.2 差別化ポイント

| 観点     | 他タスク管理アプリ | 本アプリ                                        |
| -------- | ------------------ | ----------------------------------------------- |
| 構造     | 自由設計           | 強制（ゴール→マイルストーン→タスクの3階層のみ） |
| 思考負荷 | 高い               | 低い                                            |
| UI       | 多機能・複雑       | 必要最小限                                      |
| 継続性   | ユーザー依存       | 構造で支援                                      |
| 目的志向 | 弱い               | 強い                                            |

### 4.3 UXガードレール（禁止事項）

以下の体験は絶対に提供しません。

- オンボーディングのスキップ
- ゴール未作成状態での通常利用
- ゴール直下へのタスク作成
- 完了済み要素（進捗100%）の編集
- 構造を飛び越える操作（例：タスクをマイルストーン間で移動）

## 5. ターゲットユーザー

### 5.1 公式ターゲット定義

> **「目的は持っているが、計画に落とせず行動に迷っている個人」**

### 5.2 前提条件

- 個人利用（B2C）
- 思考の構造化が苦手、または習慣化できていない
- ツールに自由度を求めるのではなく、導線と正解を求めている

### 5.3 代表ペルソナ

**高校2年生・受験生**

- 志望校は決まっている
- 何をどの順番で勉強すべきか分からない
- 計画を立てようとしても複雑になり挫折する
- 不安と焦りを感じている

### 5.4 非対象（明示）

以下の用途・ユーザーは対象外です。

- チームタスク管理
- 業務KPI管理
- 習慣トラッカー
- SNS連携
- 自由構造ノートツール

## 8. MVP方針

### 8.1 現在のフェーズ

**Phase1（MVP）** — 核となる体験の完成

### 8.2 MVP前提

| 項目           | 方針                     |
| -------------- | ------------------------ |
| ログイン       | 不要                     |
| アカウント登録 | 不要                     |
| オフライン利用 | 必須（ローカル保存のみ） |
| 通知機能       | 非対応                   |
| 対応言語       | 日本語のみ               |

### 8.3 MVP範囲外（実装しない）

- 検索機能（Phase1.5以降）
- 通知機能
- クラウド同期
- AIによる計画提案
- チーム機能
- データエクスポート/インポート
- Firebase Crashlytics等のクラッシュ分析

| 項目               | 目標                                               |
| ------------------ | -------------------------------------------------- |

- 既存タスク管理アプリと比較して継続率が高くなること
