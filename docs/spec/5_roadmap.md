## 2. はじめに

本ドキュメントは、ゴール達成型タスク管理アプリの**開発ロードマップ**を定義したドキュメントです。

以下を含みます：

- MVP（Phase 1）機能一覧
- Phase 2以降の拡張機能
- 優先度・見積もり
- リリーススケジュール（参考）

## 3. MVP（Phase 1）詳細

### 3.1 MVP リリース目標

| 項目             | 内容                             |
| ---------------- | -------------------------------- |
| **リリース時期** | 2026年6月（目標）                |
| **対応OS**       | iOS 12.0+、Android 5.0+          |
| **価格**         | 1,000円（買い切り）              |
| **主な特性**     | シンプル、迷わない、継続しやすい |

### 3.2 MVP 機能一覧

#### ✅ 実装する機能

| No  | 機能                  | 優先度 | 見積もり | 説明                             |
| --- | --------------------- | ------ | -------- | -------------------------------- |
| 1   | ゴール CRUD           | MUST   | 40h      | ゴール作成・閲覧・編集・削除     |
| 2   | マイルストーン CRUD   | MUST   | 30h      | MS作成・閲覧・編集・削除         |
| 3   | タスク CRUD           | MUST   | 35h      | タスク作成・閲覧・編集・削除     |
| 4   | タスク状態管理        | MUST   | 15h      | Todo/Doing/Done の3状態          |
| 5   | 進捗自動計算          | MUST   | 20h      | 階層別進捗の自動計算・更新       |
| 6   | ゴール一覧表示        | MUST   | 25h      | フィルター・検索・ソート機能付き |
| 7   | ゴール詳細表示        | MUST   | 20h      | MS一覧、操作パネル表示           |
| 8   | リストビュー          | MUST   | 30h      | MS→Task の階層表示               |
| 9   | ピラミッドビュー      | MUST   | 40h      | 視覚的な階層表示                 |
| 10  | カレンダービュー      | MUST   | 45h      | 期限ベースのカレンダー表示       |
| 11  | 今日のタスク表示      | MUST   | 30h      | 本日タスク横断表示               |
| 12  | ローカル保存（Hive）  | MUST   | 25h      | Hive によるローカルDB            |
| 13  | 初回オンボーディング  | MUST   | 20h      | スプラッシュ画面、説明画面       |
| 14  | UI/UX（基本設計）     | MUST   | 50h      | 画面デザイン、ウィジェット実装   |
| 15  | テスト（Unit/Widget） | MUST   | 60h      | 80%以上のテストカバレッジ        |
| 16  | バグ修正・最適化      | MUST   | 40h      | リリース前の調整                 |

**合計見積もり時間**：約 525 時間（約 13 週間、3人チーム想定）

#### ❌ 非実装機能

| No  | 機能               | 理由                                               | 計画フェーズ |
| --- | ------------------ | -------------------------------------------------- | ------------ |
| 1   | 通知機能           | MVP では不要、ユーザーが積極的に確認               | Phase 2      |
| 2   | クラウド同期       | インフラ構築コスト大、MVP では ローカルのみ        | Phase 2      |
| 3   | ユーザーアカウント | クラウド同期と同時実装                             | Phase 2      |
| 4   | AI提案機能         | 開発コスト大、ユーザーの思考負荷を減らす思想に矛盾 | Phase 3      |
| 5   | SNS連携            | MVP では個人利用のみ                               | Phase 3      |
| 6   | テーマ変更         | シンプル性を損なう、ユーザーニーズ低               | Phase 3      |
| 7   | テンプレート機能   | ユーザーの思考を奪う、哲学と矛盾                   | Phase 4      |
| 8   | 共有機能           | チーム機能の実装が必要                             | Phase 4+     |
| 9   | Web版              | 開発工数大、MVP では不要                           | Phase 4+     |

### 3.3 MVP テスト計画

| テスト種類           | 対象                  | 目標               |
| -------------------- | --------------------- | ------------------ |
| **Unit Test**        | Domain, Application層 | 80%以上カバレッジ  |

| **Integration Test** | 主要フロー            | CRUD、計算ロジック |

## 4. Phase 2（1年以内）

### 4.1 目標・背景

- MVP リリース後のユーザーフィードバック取得
- データ同期ニーズへの対応
- ユーザー数の増加に対応した機能追加

### 4.2 実装予定機能

| No  | 機能               | 優先度 | 見積もり | 説明                                  |
| --- | ------------------ | ------ | -------- | ------------------------------------- |
| 1   | クラウド同期       | HIGH   | 80h      | Firebase Firestore / Realtime DB 連携 |
| 2   | ユーザーアカウント | HIGH   | 60h      | Sign-in / Sign-up、認証               |
| 3   | 通知機能           | MEDIUM | 40h      | リマインド通知、期限前通知            |
| 4   | データバックアップ | MEDIUM | 30h      | JSON エクスポート・インポート         |
| 5   | テーマ変更         | LOW    | 20h      | ダークモード、ライトモード            |
| 6   | 複数言語対応       | LOW    | 30h      | 日本語、英語対応                      |
| 7   | バグ修正・最適化   | MEDIUM | 40h      | ユーザーフィードバック対応            |

**合計見積もり時間**：約 300 時間

### 4.3 Phase 2 の主な変更

#### クラウド同期アーキテクチャ

```
┌─────────────────────┐
│  Presentation Layer │
├─────────────────────┤
│  Application Layer  │
├─────────────────────┤
│   Domain Layer      │
├─────────────────────┤
│ Infrastructure      │
│ ┌─────────────────┐ │
│ │ Local Hive      │ │ ← キャッシュ
│ ├─────────────────┤ │
│ │ Remote Firebase │ │ ← クラウド
│ ├─────────────────┤ │
│ │ Sync Logic      │ │ ← 同期エンジン
│ └─────────────────┘ │
└─────────────────────┘
```

**同期方針**：

- ローカルファーストアプローチ
- バックグラウンド同期
- 競合解決ロジック（最終更新時刻ベース）

## 9. リスク管理

### 9.1 主要リスク

| No  | リスク               | 影響度 | 対応策                            |
| --- | -------------------- | ------ | --------------------------------- |

| 4   | クラウド連携の複雑化 | MEDIUM | 早期にPoC実施                     |
| 5   | 市場での競合増加     | HIGH   | 継続的な差別化、UI/UX向上         |
