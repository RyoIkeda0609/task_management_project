## 2. エグゼクティブサマリー

本ドキュメントは、ゴール達成型タスク管理アプリの**プロダクトコンセプト**を統一したドキュメントです。

- 何を解決するのか
- 誰に向けたアプリか
- どのような価値を提供するのか
- 他アプリとの差別化は何か

を明確に定義し、開発チーム全体での方向性の統一を目的としています。

---

## 3. プロダクトコンセプト

### 3.1 一言表現

> **「目的達成のために、今日やることが分かるアプリ」**

このアプリを使うことで、ユーザーは以下を実現できます：

- 何のために行動しているのかが明確になる
- 今日やるべきことが迷わずに分かる
- 進捗が視覚的に理解できる
- ゴール達成への自信を持つことができる

---

## 4. 解決する課題

### 4.1 既存タスク管理アプリの問題点

現在市場にあるタスク管理アプリ（Todoist、Notion、TickTick など）の多くは：

| 問題点                         | 詳細                                                 |
| ------------------------------ | ---------------------------------------------------- |
| **単発タスク管理に偏している** | 買い物、勉強、宿題などの「やることリスト」として機能 |
| **自由度が高すぎる**           | ユーザーが自分で構造を設計する必要がある             |
| **目的とタスクの関連が曖昧**   | なぜこのタスクをやるのか不明確になりやすい           |
| **継続利用されにくい**         | 構造設計の煩雑さ、動機の喪失により継続率が低い       |

### 4.2 ユーザーが実際に困っていること

**本質的課題：「目的 → 計画 → 行動」への分解が人力では難しい**

具体例：

#### ペルソナ1：受験生

- 「志望校に合格したい」という目標がある
- 「何を勉強すべきか」「いつまでに」が不明確
- 計画を立てられず、不安感が続く
- なんとなく勉強しても、目標との関連が見えない

#### ペルソナ2：ダイエット実行者

- 「今年の夏までに体重を10kg落としたい」
- 「月単位での目標」「週単位での行動」が不明確
- 毎日何をするべきか迷い続ける
- 継続できない

#### ペルソナ3：キャリア構築者

- 「志望業界での転職を実現したい」
- 「どのスキルを習得すべきか」「いつまでに」が曖昧
- 日々の学習計画が立てられない

## 6. 提供価値

### 6.1 コア価値

本アプリが提供する**3つの核となる価値**：

#### 価値1：「構造化思考の強制」

- **What**：ゴール → マイルストーン → タスク の3階層構造を唯一の正解として提供
- **Why**：ユーザーに構造設計の負荷をかけず、考える道筋を明確化
- **Impact**：迷わずに行動計画を立てられる

#### 価値2：「今日やることの明確化」

- **What**：複数ゴール配下のタスクを横断して「今日のタスク」として一覧化
- **Why**：多くのゴールを持つユーザーでも「今日やることは何か」が即座に分かる
- **Impact**：毎日の行動が明確になり、実行効率が向上

#### 価値3：「進捗の可視化」

- **What**：タスク完了に基づいた自動進捗計算（タスク → マイルストーン → ゴール）
- **Why**：ユーザーの努力が「目に見える形」で積み上がることを実感させる
- **Impact**：達成感の積み重ね、継続のモチベーション維持

### 6.2 ユーザーが得られる状態（1ヶ月後）

本アプリを継続利用したユーザーは以下の状態に変わる：

| 変化の次元   | ビフォー                       | アフター                                   |
| ------------ | ------------------------------ | ------------------------------------------ |
| **思考方法** | なんとなく、その日の気分で行動 | 「目的 → 達成条件 → 行動」を習慣的に考える |
| **意思決定** | 何をすべきか不明確、迷い続ける | 優先順位が明確、即座に行動できる           |
| **感情**     | 不安、焦り、やる気の波がある   | 自信、充実感、前進している実感             |
| **行動**     | 継続できない                   | 毎日続けられる                             |
| **成果**     | ゴール達成まで見えない         | 達成に向けた道筋が見える                   |

## 7. 差別化ポイント

### 7.1 「方向性3：シンプルさと強制力による継続性」

本アプリの差別化は、**「ユーザーが迷わない」「継続できる」アプリであること**です。

| ポイント         | 他アプリ                                   | このアプリ                                    |
| ---------------- | ------------------------------------------ | --------------------------------------------- |
| **設計の自由度** | 高い（タグ、カテゴリ、優先度を自由に設定） | 強制（ゴール → マイルストーン → タスク のみ） |
| **ユーザー負荷** | 高い（「どう使うか」をユーザーが決める）   | 低い（使い方が決まっている）                  |
| **UI複雑さ**     | 複雑（多機能、カスタマイズ豊富）           | シンプル（必要最小限の機能）                  |
| **習慣化難度**   | 難しい（自由度の高さが迷いを生む）         | 簡単（毎日同じUIで同じ操作）                  |
| **継続率**       | 低い                                       | 高い（想定値）                                |

### 7.2 具体的な差別化実装

#### 差別化1：「期限必須」による強制力

- マイルストーン・タスクの期限を必須項目に設定
- 「いつまでに」が曖昧なタスクを作らせない設計
- **結果**：ユーザーが後回しにしにくくなる

#### 差別化2：「3階層構造の唯一性」

- ゴール直下へのタスク作成禁止
- 自由な階層構造は提供しない
- **結果**：ユーザーが「どう構成するか」で迷わない

#### 差別化3：「視覚的単純性」

- テーマ変更、複数のビューモードはMVPでは提供しない
- 毎回同じシンプルなUI体験
- **結果**：習慣化しやすく、学習コストが低い

#### 差別化4：「今日のタスク」の最優先表示

- アプリ起動時のデフォルト画面は「ゴール一覧」だが、「今日のタスク」へのアクセスが常に1タップ
- 複数ゴール横断での「今日やることリスト」
- **結果**：ユーザーが毎日実行すべきことを見落とさない

## 8. コア思想

### 8.1 3つの基本原則

#### 原則1：「構造は強制、分解はユーザー」

- アプリが強制する：ゴール → マイルストーン → タスク の構造
- ユーザーが自分で考える：具体的な分解内容、期限設定
- **なぜ**：構造的な思考を習慣化させつつ、ユーザーの主体性を保つ

#### 原則2：「思考負荷を極限まで減らす」

- 「どう使うか」について考えさせない
- 「何をするか」「いつまでにするか」だけに集中させる
- **結果**：継続利用を可能にする

#### 原則3：「ゴール達成体験を最優先」

- すべての設計判断は「ユーザーがゴールを達成できるか」を基準に
- 装飾的な機能は後回し
- **例**：通知機能より、「今日のタスク」画面の使いやすさを優先

## 11. 附録：用語定義

| 用語               | 定義                                                                 |
| ------------------ | -------------------------------------------------------------------- |
| **ゴール**         | ユーザーが達成したい最終的な目的（例：「志望校に合格」）             |
| **マイルストーン** | ゴール達成のための中間目標（例：「共通テスト で○○点取得」）          |
| **タスク**         | マイルストーン達成のための具体的な行動（例：「英語の過去問を解く」） |
| **進捗**           | ゴール / マイルストーン / タスク の達成度合いを示す数値（%）         |
| **MVP**            | 最小限の機能を備えた製品版（Minimum Viable Product）                 |
| **継続率**         | アプリをインストール後、一定期間経過後も利用しているユーザーの比率   |
