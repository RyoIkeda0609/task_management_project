- 将来の機能拡張・運用に耐えるプロダクト資産の蓄積

## 2. 全体像サマリー

### 2.1 プロダクトコンセプト

> **「目的達成のために、今日やることが分かるアプリ」**

### 2.2 解決する課題

既存のタスク管理アプリは自由度が高く、
ユーザー自身が構造を設計する必要があるため、以下の問題が発生します。

- 何のためのタスクか分からなくなる
- ゴールから逆算した計画が立てられない
- 行動が継続しない

**本質的課題**：
「目的 → 計画 → 行動」への分解を人力で行うことは難しい

### 2.3 想定ユーザー（公式定義）

**目的は持っているが、計画に落とせず行動に迷っている個人**

本プロダクトは以下を前提とします。

- 個人利用（B2C）
- 思考の構造化が苦手、または習慣化できていない
- ツールに自由度を求めるのではなく、導線と正解を求めている

※ チーム利用、業務KPI管理、習慣トラッカー用途は本プロダクトの対象外とします。

### 2.4 差別化ポイント

**シンプルさと強制力による継続性**

| 観点     | 他タスク管理アプリ | 本アプリ                         |
| -------- | ------------------ | -------------------------------- |
| 構造     | 自由設計           | ゴール→マイルストーン→タスクのみ |
| 思考負荷 | 高い               | 低い                             |
| UI       | 多機能・複雑       | 必要最小限                       |
| 継続性   | ユーザー依存       | 構造で支援                       |

**仮説**：
既存の一般的なタスク管理アプリと比較して、継続率が高くなることを目指す。

### 2.5 コア思想（最上位原則）

- **構造は強制、分解はユーザー**
- **思考を奪わず、迷いを奪う**
- **ゴール達成体験を最優先する**
